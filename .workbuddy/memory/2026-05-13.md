# 2026-05-13 工作记录

## SKAND Studio 官网修复

### 问题1：logo.png 在手机端不显示
- 原因：本地有 `logo.png` 但服务器 `/opt/1panel/www/wwwroot/www.skandstudio.com/` 上没有
- 解决：已配置 SSH 密钥登录（公钥添加到服务器 `/root/.ssh/authorized_keys`），用 scp 上传 `logo.png`

### 问题2：Lightbox 打开时自定义光标消失
- 原因：自定义光标 `z-index: 10000`，lightbox `z-index: 10001`，光标被遮住；且 body 设了 `cursor: none`
- 解决：在 `showLightbox()` 中把光标 `z-index` 提升到 `10002`，`closeLightbox()` 中恢复

### 问题3：手机端打开 lightbox 后背景仍可滑动
- 原因：`body { overflow: hidden }` 在手机端无法阻止 touch 滚动
- 解决：改用 `position: fixed + top: -scrollYpx` 方案，`closeLightbox()` 时用 `window.scrollTo(0, scrollY)` 恢复位置

### 问题4：手机端希望左右滑动切换 lightbox 图片
- 解决：添加 `touchstart`/`touchend` 手势检测，滑动距离 >50px 且水平方向大于垂直方向时触发 `navigateLightbox()`

### 问题5：手机端关闭 lightbox 后页面跳回顶部
- 原因：`showLightbox()`  unconditionaly 执行 `lightboxScrollY = window.scrollY`，在 lightbox 已打开状态下切换图片时 body 已是 `position: fixed`，`window.scrollY` 为 0，覆盖了正确的滚动位置
- 解决：把 `lightboxScrollY` 赋值从 `showLightbox()` 移到点击打开的 handler 里，只在首次打开时保存；`closeLightbox()` 中用 `requestAnimationFrame` 包裹 `window.scrollTo` 以确保移动端样式重置后再滚动
- 已部署到生产服务器（2026-05-13）

### 问题6：关闭 lightbox 后页面平滑滚动回原位（"滚半天"）
- 原因：`html { scroll-behavior: smooth }` 导致 `window.scrollTo()` 带有动画
- 解决：`closeLightbox()` 中临时设置 `document.documentElement.style.scrollBehavior = 'auto'`，`scrollTo` 后立即恢复
- 已部署到生产服务器（2026-05-13）
